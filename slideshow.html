3. 系统架构设计与交互拓扑
System Architecture & Topology
⚙️ 模块化分层控制设计
  • 高交互 Web 客户端：基于 Vue 3，将微服务延迟拓扑监控与 xterm.js 实时 Linux 指令终端完美编排在大屏中。
  • 游戏控制服务端 (Express+ws)：搭载五状态闭环状态机，处理竞速全生命周期；评分器 (Scorer) 在故障被修复时对决策效率完成定量计算。
  • 安全指令代理网关 (Terminal Proxy)：防御前台执行破坏性指令。采用动态参数重写技术，严格隔离命名空间。
4. 云计算创新点：K8s-Native 沙箱与轻量化故障
Cloud Computing Innovations
☸️ 真实物理集群的低延时穿透
  • 非侵入式轻量混沌发生器：通过纯生命令（如 `tc netem` 、`dd` 和 `kill 1`），免除了部署 Chaos Mesh 等重型控制平面的巨量开销，做到了微秒级生效和 100% 零残留还原。
  • 物理拓扑指标直出：大屏中的 CPU/内存利用率和异常事件直接来源于宿主机的 `kubectl top` API，真实展现了微服务内部的各种资源危机。
🔒 安全命令重写网关
  • 绝对的命名空间边界锁定：在终端代理层使用高能过滤。所有进入后端的指令均通过正则提取，动态添加并替换命名空间 `-n chaos-game`。人类用户毫无感知，却绝对无法越权删除或操纵旁路资源。
  • 危险语法硬过滤：实时过滤管道拼接、分号跳转和背景挂起等逃逸操作，在开放调试权的同时，维持了生产级的安全水位。
5. AI 创新点：DevOps Agent 与 OODA 决策闭环
AI Innovations & Self-healing Agent
🔍 快照剪枝对比降噪算法 (Snapshot RAG)
  • 破解可观测日志泛滥难题：传统的 K8s Events 日志动辄上万字，包含大量无用信息，大模型在此环境下极易产生推理幻觉，Token 消耗呈指数级上升。
  • 差异原子裁剪：AI 引擎定时读取集群，与健康基线 (Golden Baseline) 作原子级比对。剔除一切正常的服务细节和长串 UUID。
  • 极致 Context 压缩：成功实现 **75% 的上下文裁剪**，仅向 LLM 喂入最干净的突发异常快照，模型判断确定性无限接近 100%。
🤖 完全闭环的 OODA 智能体自愈
  • Observe (观察)：持续拉取剔除杂音的集群差异快照与系统 Events 日志。
  • Orient (调整)：利用拓扑关系自动判定上游和下游的影响范围。
  • Decide (决策)：结合思维链 (CoT)，智能定位根因，决策出推荐的 kubectl 自愈脚本。
  • Action (行动)：代理自发在宿主机下发该命令，并在后续调谐环路中观察其成效，形成完备的自愈闭环。
